Curso Git

Comandos basicos git

  • $git status:
    • Verifica el estado en el que se encuentra el repositorio.
    • Muestra los cambios realizados dentro del repositorio.
  • $git diff:
    • Muestra a detalle la modificacion realizada en los archivos.
    • Despues del comando se pueden enviar como parametro rangos en los que se desea visualizar los cambios. Ej.:
      • $git diff (rama1) (rama2):
        • Si realizaste un merge entre rama1 y 2, este comando mostrara los cambios entre estas 2 ramas
  • $git add (nombre de archivo con extension que se ha modificado):
    • Se incorpora el cambio realizado y de esta forma esta listo para su commit
  • $git reset HEAD (nombre de archivo con extension):
    • Sirve para revertir la incorporacion realizada con el comando $git add.
  • $git rm (nombre de archivo a borrar del repositorio):
    • Al borrar un archivo localmente este no se borra del repositorio; con este comando se borra el archivo en el repositorio al realizar el siguiente commit. *NOTA: rm + nombre de archivo con extension sirve para eliminar un archivo localmente
  • $git commit:
    • Se envían los cambios al repositorio local. *Se requiere un resumen breve de los cambios realizados
    • $git commit -m "Título de los cambios":
      • Se utiliza para enviar un commit con que solo tenga línea de título
    • $git commit -am "Título de los cambios":
      • Desde este comando se añaden las modificaciones realizadas y a la vez se agrega un título de una sola línea.
    • $git commit --amend:
      • Sirve para realizar un cambio en el ultimo commit que se haya realizado
  • $git reset --soft HEAD~1:
    • Deshace el último commit sin borrar ningún archivo (el número 1 significa último commits)
  • $git remote:
    • Ver la lista de remotos de tu repositorio
    • $git remote -v
      • Ver dirección url en la que trabaja el repositorio
  • $git push (nombre del remoto, en este caso ORIGIN) (nombre de la rama, en este caso es MASTER):
    • Con esto se envía el codigo a GitHub
  • $git pull (nombre del remoto, en este caso ORIGIN) (nombre de la rama, en este caso es MASTER) :
    • Se sincronizan los cambios realizados por alguien mas en el repositorio.
  • $git log:
    • Se obtiene el historial de cambios realizados.
    • $git log --decorate:
      • Muestra mas detalles en el log.
    • Algunos parametros adicionales para el comando log:
      • --graph: Muestra los cambios realizados muy similar a GitHub; con una linea vertical.
      • --oneline: Muestra descripciones cortas, en una línea.
      • --all: Muestra todo.
  • $git branch:
    • Muestra las ramas de tu repositorio.
    • $git branch (nombre de una nueva rama):
      • Se crea una nueva rama del repositorio.
  • $git branch -d (nombre de la rama):
    • Se elimina la rama
    • *Borra la rama siempre y cuando no haya cambios que se puedan perder
    • Para borrar la rama no importando los cambios que se encuentren en ella se utiliza: $git branch -D (nombre de la rama)
  • $git checkout (nombre de la rama):
    • Te mueves dentro de la ramas de tu repositorio.
    • $git checkout -b (nombre de la nueva rama):
      • Crea una nueva rama y a la vez accede a ella.
  • $git merge (nombre de la rama que se desea fusionar a la rama actual):
    • Permite integrar ramas (Estando en la rama principal, se ejecuta el comando con el nombre de la rama donde se encuentran los cambios y que se desean integrar a la rama donde nos encontramos)
  • $git gc:
    • Realiza limpieza
